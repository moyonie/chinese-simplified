# 翻译风格指南

**版本**: 2.0.1  
**创建日期**: 2026-03-27  
**语言**: zh-Hans (简体中文)  
**适用**: Flarum 2.0.0-beta.8+

---

## 🎯 翻译原则

### 1. 准确性 (Accuracy)
- 忠实于原文含义，不随意增减
- 专业术语使用标准译法
- 避免歧义和误解

### 2. 一致性 (Consistency)
- 同一术语在全文中保持统一
- 遵循术语表 (GLOSSARY.md)
- 保持语气和风格一致

### 3. 简洁性 (Conciseness)
- 按钮文案简短有力（2-4 字为佳）
- 避免冗长句式
- 删除不必要的修饰词

### 4. 友好性 (Friendliness)
- 使用第二人称"你"而非"您"（更亲切）
- 语气自然，不生硬
- 错误提示要委婉

### 5. 专业性 (Professionalism)
- 符合论坛场景
- 使用行业标准术语
- 避免口语化和网络用语

## 📝 语气规范

### 按钮文案

| 场景 | 风格 | 示例 |
|------|------|------|
| 确认操作 | 简洁动词 | 保存、取消、删除、编辑 |
| 危险操作 | 明确警示 | 删除账号、永久删除 |
| 积极操作 | 鼓励语气 | 开始讨论、发起对话 |
| 导航操作 | 清晰指向 | 返回首页、查看全部 |

### 提示信息

| 类型 | 风格 | 示例 |
|------|------|------|
| 成功提示 | 肯定语气 | "设置已保存"、"私信已发送" |
| 错误提示 | 委婉说明 + 解决方案 | "网络连接失败，请检查网络设置" |
| 警告提示 | 明确后果 | "此操作不可撤销" |
| 确认提示 | 清晰询问 | "确定要删除此讨论吗？" |

### 错误信息模板

```
[问题描述] + [原因说明（可选）] + [解决方案（可选）]

示例:
- "会话已过期，请重新登录"
- "权限不足，请联系管理员"
- "网络连接失败，请检查网络设置"
```

## 🔤 文字规范

### 标点符号

| 符号 | 使用规则 | 示例 |
|------|---------|------|
| ， | 中文逗号，用于句子停顿 | 保存更改，取消操作 |
| 。 | 中文句号，用于句子结束 | 设置已保存。 |
| ？ | 中文问号，用于疑问句 | 确定要删除吗？ |
| ！ | 中文感叹号，用于强调 | 操作成功！ |
| ： | 中文冒号，用于提示 | 注意：此操作不可撤销 |
| …… | 中文省略号（6 点） | 加载中…… |
| （） | 中文括号，用于补充说明 | 用户名（必填） |

**注意**: 
- 完整句子末尾使用标点
- 按钮文案、短标题可不加标点
- 省略号用"……"而非"..."

### 空格使用

| 场景 | 规则 | 示例 |
|------|------|------|
| 中英文混排 | 中英文之间加空格 | 使用 PHP 8.1 |
| 数字 + 单位 | 数字和单位之间不加空格 | 5 分钟、10 个帖子 |
| 变量占位符 | 占位符与中文之间加空格 | '{count} 条回复' |
| 标点符号 | 中文标点后不加空格 | 保存，取消 |

### 大小写

| 场景 | 规则 | 示例 |
|------|------|------|
| 专有名词 | 保持原有大小写 | Flarum, PHP, MySQL, GitHub |
| 句子开头 | 中文无需大写 | 设置已保存 |
| 强调文字 | 使用**加粗**而非大写 | **重要提示** |

## 📋 翻译规范

### 变量占位符

**格式**: `{variable}`

**规则**:
1. 保持占位符名称不变
2. 占位符与中文之间加空格
3. 确保占位符在译文中出现

✅ 正确:
```yaml
reply_count: '{count} 条回复'
edited_tooltip: '由 {user} 于 {time} 编辑'
```

❌ 错误:
```yaml
reply_count: '{count}条回复'  # 缺少空格
edited_tooltip: '由用户于时间编辑'  # 丢失占位符
```

### 复数形式 (ICU MessageFormat)

**格式**: `'{count, plural, one{单数形式} other{复数形式}}'`

**示例**:
```yaml
# 英文
likes: '{count, plural, one{Like} other{Likes}}'

# 中文（通常不需要复数变化）
likes: '{count} 人点赞'
```

**注意**: 中文名词通常无复数形式，直接用数量词即可。

### HTML 标签

**规则**:
1. 保持 HTML 标签不变
2. 只翻译标签内的文本
3. 注意标签嵌套关系

✅ 正确:
```yaml
welcome_message: '欢迎加入 <strong>{forum}</strong>'
```

❌ 错误:
```yaml
welcome_message: '<strong>欢迎加入 {forum}</strong>'  # 改变了标签位置
```

## 🎨 场景化翻译指南

### 删除确认

**模板**: "确定要 [操作] 吗？[后果说明]"

**示例**:
- "确定要删除此讨论吗？此操作不可撤销。"
- "确定要删除账号吗？所有数据将被永久删除。"

### 错误提示

**模板**: "[问题] + [原因（可选）] + [解决方案]"

**示例**:
- "网络连接失败，请检查网络设置"
- "会话已过期，请重新登录"
- "权限不足，请联系管理员"

### 成功提示

**模板**: "[操作] 已 [完成状态]"

**示例**:
- "设置已保存"
- "私信已发送"
- "密码已更改"

### 加载状态

**模板**: "[操作] 中..."

**示例**:
- "加载中..."
- "保存中..."
- "发送中..."

## ⚠️ 常见错误

### 1. 术语不统一

❌ 错误:
```yaml
nav:
  messages: 消息  # 此处用"消息"
  
flarum-messages:
  nav:
    messages: 私信  # 此处用"私信"
```

✅ 正确:
```yaml
nav:
  messages: 私信  # 统一用"私信"
  
flarum-messages:
  nav:
    messages: 私信
```

### 2. 标点混用

❌ 错误:
```yaml
delete_confirmation: 确定要删除吗？  # 中文句子用英文问号
```

✅ 正确:
```yaml
delete_confirmation: 确定要删除吗？  # 使用中文问号
```

### 3. 空格不规范

❌ 错误:
```yaml
version_info: 'Flarum 版本{version}'  # 缺少空格
```

✅ 正确:
```yaml
version_info: 'Flarum 版本 {version}'  # 中英文之间加空格
```

### 4. 语气生硬

❌ 错误:
```yaml
permission_denied: 你没有权限  # 过于直接
```

✅ 正确:
```yaml
permission_denied: 权限不足  # 委婉表达
```

### 5. 丢失占位符

❌ 错误:
```yaml
minutes_ago: '分钟前'  # 丢失{count}
```

✅ 正确:
```yaml
minutes_ago: '{count} 分钟前'  # 保留占位符
```
